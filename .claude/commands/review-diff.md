# Review Diff

Use this slash-command to review a local diff or PR.

1. Read repo instructions (`AGENTS.md`, `CLAUDE.md`) and the relevant
   SPEC if any.
2. Inspect the diff before summarizing.
3. Prioritize bugs, regressions, missing tests, and broken repo
   contracts.
4. Lead with findings ordered by severity (blocking → advisory → nit).
5. If no findings exist, say so and list residual risk.

Use the `code-review` skill.
