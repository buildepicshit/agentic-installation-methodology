# Orient

Use this slash-command before planning or editing.

1. Read `AGENTS.md`.
2. Read `WORKFLOW.md`, `CLAUDE.md` (if present), `STATUS.md`, and
   linked docs.
3. Read `agents/OPERATING_MODEL.md` for the v1 spec-first lifecycle.
4. Run `git status --short --branch`.
5. Identify current branch, local changes, likely files, and the
   verification gate.
6. Report verified facts, risks, and open questions.

Use the `repo-orientation` skill.
