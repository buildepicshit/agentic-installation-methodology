# Spec Evidence

Use this slash-command after substantial work or incident resolution
to capture durable lessons as spec-evidence candidates.

1. Review the Completion Report and verification output of the SPEC.
2. Identify durable lessons useful to future agents (patterns,
   pitfalls, surprises).
3. Exclude anything already covered by repo docs or `agents/` policy.
4. Record each candidate with: claim, scope, evidence, confidence,
   conflicts with existing rules, suggested SPEC or delivery-authority
   route.
5. Do NOT write durable shared rules directly from an agent session.
   Candidates flow into a follow-on spec or owner review.

Use the `spec-evidence-governance` skill.
